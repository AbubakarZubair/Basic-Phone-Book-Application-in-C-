# 🖥️ How to Run PhoneBook Pro in VSCode

This guide walks you through setting up and running **PhoneBook Pro** from scratch on Windows, macOS, or Linux using Visual Studio Code.

---

## 📋 Prerequisites

Make sure you have the following installed before you begin:

| Tool | Version | How to Check |
|------|---------|--------------|
| **Python** | 3.9 or higher | `python --version` |
| **G++ (GCC)** | Any recent version | `g++ --version` |
| **VSCode** | Latest | [Download](https://code.visualstudio.com/) |
| **Git** | Any | `git --version` |

### Installing G++

**Windows:**
```powershell
# Option 1: MSYS2 (recommended)
# Download from https://www.msys2.org/ then run:
pacman -S mingw-w64-x86_64-gcc

# Option 2: Using winget
winget install -e --id GnuWin32.Make
```

**macOS:**
```bash
xcode-select --install     # installs clang (compatible)
# or: brew install gcc
```

**Linux (Ubuntu / Debian):**
```bash
sudo apt update && sudo apt install g++ build-essential -y
```

## 🐍 Step 3 — Create the Virtual Environment

Open the **VSCode Integrated Terminal** (`Ctrl + `` ` `` ` or **View → Terminal**):

**macOS / Linux:**
```bash
python3 -m venv venv
source venv/bin/activate
```

**Windows (PowerShell):**
```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1

# If you get an execution policy error:
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
.\venv\Scripts\Activate.ps1
```

**Windows (Command Prompt):**
```cmd
python -m venv venv
venv\Scripts\activate.bat
```

You should see `(venv)` at the start of your terminal prompt.

---

## 📦 Step 4 — Install Python Dependencies

With the venv active:

```bash
pip install -r backend/requirements.txt
```

This installs:
- `fastapi` — the web framework
- `uvicorn` — the ASGI server
- `pydantic` — data validation

---

## 🔨 Step 5 — Start the API Server

```bash
uvicorn backend.api:app --reload --port 8000
```

**What happens automatically:**
1. Python finds `backend/phonebook.cpp`
2. G++ compiles it into `backend/phonebook.so` (the shared library)
3. Python loads the `.so` via `ctypes`
4. FastAPI starts listening on port 8000

You should see output like:
```
🔨 Compiling phonebook.cpp → phonebook.so …
✅ Compilation successful.
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

> **Tip:** `--reload` means the server auto-restarts when you edit Python files. 
> The C++ file is only recompiled if it's newer than the `.so`.

## 🌐 Step 6 — Open the Frontend

**Option A — Just open the file (simplest):**
1. In VSCode Explorer, right-click `frontend/index.html`
2. Select **Open with Live Server** (if you have the Live Server extension)

OR simply double-click `frontend/index.html` to open it in your browser.

**Option B — Serve it properly (avoids CORS issues on some browsers):**
```bash
# In a second terminal (keep the API running in the first):
python -m http.server 3000 --directory frontend
```
Then visit: **http://localhost:3000**

## 🐛 Troubleshooting

### ❌ `g++: command not found`
Install G++ for your OS (see Prerequisites section above).  
On Windows, make sure MSYS2's `bin` folder is in your PATH.

### ❌ `ModuleNotFoundError: No module named 'fastapi'`
Your venv isn't active. Run:
```bash
source venv/bin/activate    # Mac/Linux
.\venv\Scripts\activate     # Windows
```

### ❌ `OSError: cannot load shared object`
The `.so` failed to compile. Check:
```bash
g++ --version   # should print a version
```
Then try manually:
```bash
g++ -O2 -shared -fPIC -DLIB_BUILD -std=c++17 backend/phonebook.cpp -o backend/phonebook.so
```
Read the error output carefully.

### ❌ Frontend shows "OFFLINE"
Make sure the API server is running (`uvicorn backend.api:app --reload`).  
Check that port 8000 is not blocked by a firewall or used by another app.

### ❌ CORS error in browser console
This shouldn't happen (CORS is enabled for all origins in `api.py`).  
If it does, try serving the frontend with `python -m http.server 3000` instead of opening the file directly.

## 🚀 Running in Production (Optional)

To deploy without the `--reload` flag:

```bash
uvicorn backend.api:app --host 0.0.0.0 --port 8000 --workers 1
```

> **Note:** Use `--workers 1` because the C++ store is in-process memory (not shared between workers). For multi-worker persistence, replace the array store with SQLite.
